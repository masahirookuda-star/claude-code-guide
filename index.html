Claude Code は何が違うのか

ChatGPT や Claude.ai との最大の違いは「自律的に動く」こと。ファイルを読み書きし、コマンドを実行し、外部サービスと連携し、スケジュールに従って無人で動き続けます。

Mac の場合

Node.js をインストール

公式サイト（nodejs.org）から LTS版をダウンロードしてインストール。または Homebrew を使う場合は以下のコマンドを実行。

bash # Homebrew 経由でインストール（Homebrew がない場合は brew.sh から先にインストール） brew install node # バージョン確認 node --version # v18以上であればOK

Windows の場合

⚠️
前提：Windows の場合は WSL2（Windows Subsystem for Linux）を使って Linux 環境上で動かす方法が最も安定しています。Powershell から直接でも動きますが、自動化スクリプトなどは WSL2 推奨。
1

WSL2 + Ubuntu をセットアップ

PowerShell を管理者権限で開いて実行：

PowerShell wsl --install # 再起動後、Microsoft Store から Ubuntu をインストール
2

Ubuntu（WSL2）内で Node.js をインストール

bash (WSL2) curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash - sudo apt-get install -y nodejs node --version # v20以上を確認
3

Claude Code をインストール・起動

bash (WSL2) npm install -g @anthropic-ai/claude-code claude # ブラウザが開いてログイン画面が表示される
4

（オプション）Windows Terminal を使うと快適

Microsoft Store から Windows Terminal をインストールすると、WSL2 の操作が格段に快適になります。タブ切り替え、フォント設定、色テーマなどをカスタマイズできます。

Core Concept 1
Memory システム
Claude Code は会話をまたいで「記憶」できる
🧠

Memory とは何か

通常のAIは会話が終わるとすべて忘れます。Claude Code のMemoryは、重要な情報をファイルとして永続保存し、次のセッションでも自動的に読み込みます。

保存先

# Mac ~/.claude/projects/[プロジェクト]/memory/ ├── MEMORY.md # インデックス（自動ロード） ├── user_profile.md # ユーザー情報 ├── feedback_xxx.md # フィードバック記録 └── reference_xxx.md # 参照情報

4つのメモリタイプ

user ユーザーの役割・好み・知識
feedback 「こうしてほしい」指示の記録
project 進行中の業務・意思決定
reference 外部リソースへのポインタ

メモリファイルの構造

markdown --- name: フィードバック：コードはコメントを最小限にする description: コードコメントの指針。コード作成時に必ず参照。 type: feedback --- コード中のコメントは原則不要。変数名・関数名で意味が伝わるように書く。 **Why:** 「コメントがないほうが読みやすい」との好み。 **How to apply:** コード生成時にコメントを入れない。
💡
使い方：会話中に「〇〇を覚えておいて」と言うだけで自動的にメモリファイルに保存されます。次回のセッションから自動的に読み込まれ、前回の指示を繰り返す必要がなくなります。
Core Concept 1b
CLAUDE.md — 常時読み込まれる指示書
すべてのセッションに適用される「AIへの取扱説明書」

~/.claude/CLAUDE.md に書いた内容は、Claude Code を起動するたびに必ず読み込まれます。「自分のことを常に知っておいてほしいこと」をここに書きます。

markdown（~/.claude/CLAUDE.md） # 自分のプロフィール - 名前：山田太郎（〇〇株式会社 代表取締役） - 目標：営業効率を3倍にする ## よく使うフォルダ - 業務資料：~/work/docs/ - スクリプト：~/work/scripts/ ## 行動原則 - コードにコメントは入れない - 返答は短く要点のみ - 変更前に必ず確認する

CLAUDE.md に書くべき内容

1

自己紹介・役割・目標

Claudeが「誰に対して答えているか」を理解させる

2

よく使うパスやプロジェクト構成

ファイルの場所を毎回説明しなくて済む

3

行動ルール・禁止事項

「〇〇はしないで」「必ず〇〇する」などの繰り返し指示をまとめる

Core Concept 2
Skills（スラッシュコマンド）
繰り返し使う「作業テンプレート」を1コマンドで呼び出す
⚡

Skills とは何か

毎回同じ長いプロンプトを書く代わりに、/skill名 と入力するだけで複雑な作業を実行できる仕組み。チームで共有することも可能。

Skill ファイルの場所

# グローバル（全プロジェクトで使える） ~/.claude/skills/my-skill.md # プロジェクト専用 .claude/skills/my-skill.md

呼び出し方

# チャットに入力するだけ /digest 朝版 /sns /review /x-post

Skill ファイルの書き方

markdown（~/.claude/skills/morning-report.md） --- name: morning-report description: 毎朝の業務サマリーを生成する --- 以下の手順で今日の業務サマリーを作成してください： 1. ~/work/tasks/today.md を読み込む 2. 昨日の完了タスクを確認する（git logで） 3. 今日やるべきことを優先度順に整理する 4. Slackに投稿できる形式でサマリーを出力する 出力形式： - 3行以内で要点をまとめる - 絵文字は使わない

実際に使っているスキル例

📰

朝版・夕版のニュースダイジェストを生成。業界トレンドを自動収集して要約。

🐦

🎓

/training

研修コンテンツを自動生成。テーマを渡すと資料の構成と内容を一気に作成。

📝

/note

note記事の下書きを生成。テーマを渡すと構成から本文まで一気に作成。

💼

Core Concept 3
Rules（ルール設定）
ドメインごとに異なる「判断基準」をAIに教える
📋

Rules とは何か

Memory がユーザー情報を記憶するのに対し、Rules は「このカテゴリの仕事はこう動け」という業務ルールを定義します。

ファイル構造

~/.claude/rules/ ├── content.md # SNS・コンテンツ作成ルール ├── ops.md # 業務・運用系ルール ├── consulting.md # 営業・提案系ルール └── dev.md # 開発・コードルール

書き方の例

markdown（rules/content.md） # SNSコンテンツルール ## トーン - 一人称は「私」 - 体言止めを多用する - 専門用語は使わず平易な言葉で ## X投稿の禁止事項 - 絵文字を3個以上使わない - 140文字を超えるスレッドは3連投まで ## note記事 - 見出しは最大H2まで - 冒頭3行で結論を述べる
💡
フィードバックループが重要：Claudeが間違った動きをしたとき、「なぜ修正が必要だったか」を分析してRulesに追記する習慣をつけると、同じミスが繰り返されなくなります。
→ 気づき → ルール追記 → 定期的な見直しのサイクルが品質を上げる鍵。
Core Concept 4
Hooks（自動トリガー）
特定のイベントで自動的にスクリプトを実行する
🔗

Hooks とは何か

「Claudeが返答したとき」「ファイルを書き込む前」など、特定のタイミングでシェルコマンドを自動実行する仕組み。通知・ログ記録・セキュリティチェックなどに使います。

タイミングの種類

タイミング説明代表的な使いみち
Stop Claudeが返答完了したとき 完了通知・セッションログ記録
PreToolUse ツール（ファイル書込など）実行前 APIキー漏洩チェック・確認
PostToolUse ツール実行後 自動テスト・差分記録
PreCompact コンテキスト圧縮前 会話ログのバックアップ

Hooksの設定（settings.json）

json（~/.claude/settings.json） { "hooks": { "Stop": [ { "type": "command", "command": "bash ~/.claude/hooks/notify.sh" } ], "PreToolUse": [ { "matcher": "Write", "hooks": [{ "type": "command", "command": "bash ~/.claude/hooks/secret-check.sh" }] } ] } }

実際のフックスクリプト例（Mac）

bash（notify.sh） #!/bin/bash # 作業完了をmacOSに通知する osascript -e 'display notification "作業完了しました" with title "Claude Code" sound name "Glass"'
bash（secret-check.sh） #!/bin/bash # ファイルにAPIキーが含まれていたら書き込みを中断 if grep -qE '(sk-|AIza|AKIA)[A-Za-z0-9]+' "$CLAUDE_TOOL_INPUT_PATH" 2>/dev/null; then echo "SECRET DETECTED" >&2 exit 1 fi
⚠️
Windows（WSL2）での通知：macOSの osascript は使えません。Windows では powershell.exe -Command "..." を使って Windows 通知を出すか、echo でターミナルに表示するだけでもOKです。
Core Concept 5
MCP（外部サービス連携）
Claude から Slack・Gmail・カレンダー・会計ツールを直接操作する

MCP（Model Context Protocol）は、Claude に外部サービスの「ツール」を追加する仕組みです。Slack にメッセージを送ったり、Googleカレンダーを読んだりを、Claude の会話の中で直接できるようになります。

Core Concept 6
セキュリティ設定
強力なツールだからこそ、権限の管理が重要

ブロックすべき操作（deny）

DENY

~/.ssh/** / ~/.aws/** の読み取り

SSHキー・AWSクレデンシャルの漏洩を防ぐ

DENY

~/.bashrc / ~/.zshrc の編集

シェル設定の無断書き換えを防ぐ

DENY

settings.json 自体の編集

AIが自分の権限設定を書き換えることを防ぐ

DENY

nc（netcat）/ wget

不要なネットワーク操作ツールをブロック

明示的に許可する操作（allow）

ALLOW

git 系コマンド全般

開発作業の基本。push は都度確認を推奨

ALLOW

python3 / node / npm

スクリプト実行・開発作業に必要

ALLOW

curl

外部API呼び出しに必要。wgetはDENY推奨

ALLOW

ls / grep / ls 等の読み取り系

コード探索・調査に必要な基本コマンド

セキュリティ設定の実装例

json（~/.claude/settings.json の一部） { "permissions": { "deny": [ "Read(~/.ssh/**)", "Read(~/.aws/**)", "Read(~/.gnupg/**)", "Edit(~/.bashrc)", "Edit(~/.zshrc)", "Edit(~/.claude/settings.json)", "Bash(nc *)", "Bash(wget *)" ], "allow": [ "Bash(git *)", "Bash(python3 *)", "Bash(node *)", "Bash(npm *)", "Bash(curl *)" ] } }
🛡️
APIキーの管理：APIキーは絶対にチャットに貼り付けないこと。代わりにファイルに書いてパスを伝える、または環境変数（~/.zshrc や .env ファイル）で管理する。

Claude Code Remote

claude.ai の Web版から「スケジュール実行」を設定できます。毎時・毎日・毎週など、指定したプロンプトを自動で実行し続けます。PCが起動していなくてもクラウドで動きます。

# /schedule コマンドで設定 /schedule # → スケジュール設定画面が開く
⏰

macOS launchd / Windowsタスクスケジューラ

PCが起動している間に定期実行する方法。claudeコマンドをシェルスクリプトから呼び出すことで完全自動化できます。

# Mac: launchd plist で毎朝7時に実行 StartCalendarInterval: Hour: 7 Minute: 0 # スクリプト内で claude コマンドを呼ぶ claude --dangerously-skip-permissions -p "..."
🛡️
--dangerously-skip-permissions について：無人実行時に許可プロンプトをスキップするフラグです。セキュリティ設定（deny/allow）をしっかり設定した上で使ってください。本番データや外部サービスへの書き込みを伴う自動化では特に注意が必要です。
