Step 1 — Node.js と Claude Code のインストール

Chapter 1 · 環境構築
🍎 Mac
1

Homebrew をインストール（未導入の場合）

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
2

Node.js をインストール

brew install node node --version # v18以上を確認
3

Claude Code をインストール

npm install -g @anthropic-ai/claude-code
🪟 Windows（WSL2推奨）
1

WSL2 + Ubuntu をインストール

PowerShellを管理者権限で開く

wsl --install # 再起動後にUbuntu起動
2

Node.js をインストール（Ubuntu内）

curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash - sudo apt-get install -y nodejs
3

Claude Code をインストール

npm install -g @anthropic-ai/claude-code
💡
Windows Terminal 推奨：Microsoft Store から「Windows Terminal」をインストールするとWSL2操作が快適になります（タブ切替・フォント・テーマ設定可）。

Step 4 — セキュリティ設定（settings.json）

Chapter 2 · コア機能の設定

Claude Code は強力なので、最初にセキュリティ設定を入れておく。以下をそのままコピペして ~/.claude/settings.json に貼り付けるだけ。

📋 コピペOK — settings.json（セキュリティ設定）
Cursor/VS Codeで貼り付け推奨
{ "permissions": { "deny": [ "Read(~/.ssh/**)", "Read(~/.aws/**)", "Read(~/.gnupg/**)", "Read(~/.npmrc)", "Edit(~/.bashrc)", "Edit(~/.zshrc)", "Edit(~/.claude/settings.json)", "Bash(nc *)", "Bash(wget *)" ], "allow": [ "Bash(git *)", "Bash(python3 *)", "Bash(node *)", "Bash(npm *)", "Bash(curl *)", "Bash(ls *)", "Bash(grep *)" ] }, "defaultMode": "auto" }
🚨
必ずCursor/VS Codeで編集：ターミナルにコピペすると改行が入りJSONが壊れます。エディタで直接開いて貼り付けてください。
✅
確認コマンド：
python3 -m json.tool ~/.claude/settings.json
エラーが出なければJSON正常。

Step 5 — Hooks の設定（完了通知・シークレット検知）

Chapter 2 · コア機能の設定
1

hooksフォルダを作成して一括セットアップ

以下のコマンドをターミナルに貼り付けるだけですべてのhookが設定されます。

📋 コピペOK — Hooks 一括セットアップ（Mac）
ターミナルに貼り付けて実行
mkdir -p ~/.claude/hooks # 完了通知スクリプト cat > ~/.claude/hooks/notify.sh << 'EOF' #!/bin/bash osascript -e 'display notification "作業完了しました" with title "Claude Code" sound name "Glass"' EOF # シークレット検知スクリプト cat > ~/.claude/hooks/secret-check.sh << 'EOF' #!/bin/bash INPUT=$(cat) if echo "$INPUT" | grep -qiE 'password|secret|token.*=|key.*='; then echo "WARNING: 機密情報が含まれている可能性があります" >&2 fi echo "$INPUT" EOF chmod +x ~/.claude/hooks/notify.sh ~/.claude/hooks/secret-check.sh echo "Hooks setup complete!"
📋 コピペOK — Hooks 一括セットアップ（Windows WSL2）
WSL2ターミナルに貼り付けて実行
mkdir -p ~/.claude/hooks cat > ~/.claude/hooks/notify.sh << 'EOF' #!/bin/bash powershell.exe -Command " Add-Type -AssemblyName System.Windows.Forms [System.Windows.Forms.MessageBox]::Show('作業完了しました','Claude Code') " 2>/dev/null || echo "Claude Code: 作業完了" EOF chmod +x ~/.claude/hooks/notify.sh echo "Hooks setup complete!"
2

settings.json の hooks セクションに追記（Cursor/VS Codeで）

📋 コピペOK — settings.json に追記する hooks 設定
"hooks": { "Stop": [ { "type": "command", "command": "bash ~/.claude/hooks/notify.sh" } ] }

Step 6 — Skills（スラッシュコマンド）の作成

Chapter 2 · コア機能の設定

よく使う作業をSkillとして定義。/skill名 と打つだけで複雑な指示を自動実行。以下を一括作成するスクリプトをコピペ。

📋 コピペOK — Skills フォルダと6種類のSkillを一括作成
ターミナルに貼り付けて実行
mkdir -p ~/.claude/skills # SNS投稿案生成 cat > ~/.claude/skills/sns-post.md << 'EOF' --- name: sns-post description: テーマからX投稿案を3案生成する --- 以下の条件でX投稿案を3案生成してください。 テーマ：{引数として渡されたテキスト} 条件：140文字以内 / 数字を1つ以上含める / 絵文字2個まで 3案に【A案】【B案】【C案】をつけて提示してください。 EOF # 議事録生成 cat > ~/.claude/skills/minutes.md << 'EOF' --- name: minutes description: 会議メモから議事録を生成する --- 以下のメモから議事録を生成してください。 フォーマット：①決定事項 ②アクションアイテム（担当者・期限付き） ③次回議題 EOF # コードレビュー cat > ~/.claude/skills/review.md << 'EOF' --- name: review description: 変更したコードをレビューする --- git diff で変更されたコードをすべて確認し、以下の観点でレビューしてください： ①セキュリティリスク ②バグの可能性 ③パフォーマンス ④可読性 問題点は重大度（高/中/低）付きで列挙してください。 EOF # 朝のサマリー cat > ~/.claude/skills/morning.md << 'EOF' --- name: morning description: 今日の業務サマリーを生成する --- ~/work/tasks/today.md を読み込み、以下の形式でサマリーを生成してください： ①最優先タスク（1件） ②完了済み件数 ③今日の完了目標数 3行以内で要点のみ出力してください。 EOF # note記事下書き cat > ~/.claude/skills/note.md << 'EOF' --- name: note description: noteの記事下書きを生成する --- 以下のテーマでnote記事の下書きを生成してください。 テーマ：{引数} 構成：冒頭3行で結論 → 背景・問題提起 → 解決策3点 → まとめ 文体：です・ます調、専門用語なし、2000〜3000字 EOF # 商談準備 cat > ~/.claude/skills/prep.md << 'EOF' --- name: prep description: 商談相手の情報を調査して準備資料を生成する --- 次の商談相手について調査してください：{引数（会社名・担当者名）} 調査内容：①会社概要・規模 ②最近のニュース・動向 ③想定される課題・ニーズ ④話すべき提案ポイント 2〜3点 EOF echo "Skills setup complete! 6 skills created." echo "Usage: claude を起動して /sns-post テーマ など入力"

Step 9 — 定期自動実行の設定

Chapter 3 · 外部連携と自動化
1

実行スクリプトを作成（Mac/Windows共通）

📋 コピペOK — 汎用実行スクリプト（~/work/scripts/run-claude.sh）
#!/bin/bash LOG=~/work/logs/claude_auto.log mkdir -p ~/work/logs echo "[$(date '+%Y-%m-%d %H:%M')] 開始" >> "$LOG" claude --dangerously-skip-permissions -p " # ここに実行したいプロンプトを書く（例：以下は朝のナレッジ収集） 今日の組織マネジメント関連の最新情報を5件収集し、 各100文字以内で要約してください。 Slack MCPを使って #knowledge チャンネルに投稿してください。 投稿冒頭：「本日のナレッジ ($(date +%Y/%m/%d))」 " >> "$LOG" 2>&1 echo "[$(date '+%Y-%m-%d %H:%M')] 完了" >> "$LOG"
chmod +x ~/work/scripts/run-claude.sh
🍎 Mac（launchd）— 毎朝7時に自動実行
📋 コピペOK — plist ファイル
<?xml version="1.0" encoding="UTF-8"?> <!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd"> <plist version="1.0"><dict> <key>Label</key> <string>com.yourname.claude-auto</string> <key>ProgramArguments</key> <array> <string>/bin/bash</string> <string>/Users/[あなたのユーザー名]/work/scripts/run-claude.sh</string> </array> <key>StartCalendarInterval</key> <dict> <key>Hour</key><integer>7</integer> <key>Minute</key><integer>0</integer> </dict> <key>StandardOutPath</key> <string>/Users/[あなたのユーザー名]/work/logs/launchd.log</string> <key>StandardErrorPath</key> <string>/Users/[あなたのユーザー名]/work/logs/launchd_err.log</string> </dict></plist>
# [yourname]を自分のユーザー名に書き換えてから実行 cp /path/to/plist ~/Library/LaunchAgents/com.yourname.claude-auto.plist launchctl load ~/Library/LaunchAgents/com.yourname.claude-auto.plist
🪟 Windows — タスクスケジューラに登録
📋 コピペOK — PowerShell（管理者権限）
$action = New-ScheduledTaskAction ` -Execute "wsl.exe" ` -Argument "bash ~/work/scripts/run-claude.sh" $trigger = New-ScheduledTaskTrigger ` -Daily -At "07:00" $settings = New-ScheduledTaskSettingsSet ` -RunOnlyIfNetworkAvailable Register-ScheduledTask ` -TaskName "ClaudeAutoRun" ` -Action $action ` -Trigger $trigger ` -Settings $settings ` -RunLevel Highest ` -Force Write-Host "タスク登録完了"

よくあるトラブルと対処法

❌ command not found: claude

npm config get prefix # 表示パス/bin を ~/.zshrc に追加： export PATH="$HOME/.npm-packages/bin:$PATH"

❌ settings.json が読み込まれない

# JSON形式を確認 python3 -m json.tool ~/.claude/settings.json

❌ Hook が動かない

chmod +x ~/.claude/hooks/*.sh # 手動テスト bash ~/.claude/hooks/notify.sh

❌ launchd が動かない（Mac）

launchctl unload ~/Library/LaunchAgents/*.plist launchctl load ~/Library/LaunchAgents/*.plist cat ~/work/logs/launchd_err.log

❌ MCP ツールが見えない

claude を再起動。settings.json の mcpServers セクションが正しいか確認。JSON形式も確認。

❌ Windows 日本語化け

sudo locale-gen ja_JP.UTF-8 sudo update-locale LANG=ja_JP.UTF-8

実装完了後にやること

設定を整えたら「使いながら育てる」が重要です。Claudeが間違えたとき → なぜ間違えたかを分析 → rules/ や CLAUDE.md に追記 → 同じミスが繰り返されなくなる。このサイクルが数ヶ月後に大きな差を生みます。

最終チェックリスト

  • Node.js と Claude Code をインストールした
  • claude.ai の Pro 以上のアカウントでログインした
  • ~/.claude/CLAUDE.md にプロフィール・ルールを書いた
  • settings.json にセキュリティ設定（deny/allow）を追加した
  • 完了通知フック（Stop hook）を設定してテストした
  • よく使う作業を Skill として6種類定義した
  • 「コピペ初期メモリ」を claude に貼り付けて記憶させた
  • Slack / Gmail / Googleカレンダーのどれかを MCP 接続した
  • 定期実行スクリプトを1本作成してスケジューラに登録した
  • Cursor をインストールして Claude Code と使い分けを把握した
🎉
全項目チェックできたら完了！あとは Rules にフィードバックを蓄積し続けることで、Claude Code はどんどん「自分のことを知っているAI」に育っていきます。
