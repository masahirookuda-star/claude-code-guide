Step 1 — Node.js と Claude Code のインストール

Chapter 1 · 環境構築
🍎 Mac
1

Homebrew をインストール（未インストールの場合）

bash# ターミナルを開いて貼り付け /bin/bash -c "$(curl -fsSL \ https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
2

Node.js をインストール

brew install node node --version # v18以上を確認
3

Claude Code をインストール

npm install -g @anthropic-ai/claude-code
🪟 Windows（WSL2推奨）
1

WSL2 + Ubuntu をインストール

PowerShell を管理者権限で開く

PowerShellwsl --install # 再起動後 Ubuntu が起動する
2

Ubuntu で Node.js をインストール

bash (WSL2)curl -fsSL https://deb.nodesource.com/setup_20.x \ | sudo -E bash - sudo apt-get install -y nodejs
3

Claude Code をインストール

npm install -g @anthropic-ai/claude-code
💡
Windows Terminal の導入を推奨：Microsoft Store から「Windows Terminal」をインストールすると WSL2 の操作が快適になります。タブ切替・フォント設定・カラーテーマが使えます。
🔑

Step 3 — CLAUDE.md の作成

Chapter 1 · 環境構築

CLAUDE.md は Claude Code を起動するたびに必ず読み込まれる「AIへの取扱説明書」です。自分のことを毎回説明しなくて済むようになります。

1

ファイルを作成する

mkdir -p ~/.claude nano ~/.claude/CLAUDE.md # または Cursor / VS Code で開いて編集
2

以下のテンプレートを参考に書く

markdown# 自分のプロフィール - 名前：山田太郎（株式会社〇〇 代表取締役） - 役割：営業・マーケティング全般を担当 - 目標：月の商談数を2倍にする ## よく使うフォルダ - 業務ファイル：~/work/docs/ - スクリプト：~/work/scripts/ ## 行動原則 - 返答は短く要点のみ。長文にしない - ファイルを削除する前に必ず確認する - APIキー等の機密情報はチャットに出力しない ## 禁止事項 - settings.json を勝手に書き換えない - .ssh や .aws 配下を読み込まない
✅
確認方法：claude を再起動して「私の名前は？」と聞くと、CLAUDE.md の内容を反映した返答が来ます。

Step 4 — セキュリティ設定（settings.json）

Chapter 2 · コア機能の設定

Claude Code は強力なツールです。最初にセキュリティ設定を入れておくことで、意図しないファイル操作やデータ漏洩を防ぎます。

1

~/.claude/settings.json を Cursor / VS Code で編集

コピペの改行問題を避けるため、必ずエディタで直接編集してください。

json{ "permissions": { "deny": [ "Read(~/.ssh/**)", "Read(~/.aws/**)", "Read(~/.gnupg/**)", "Edit(~/.bashrc)", "Edit(~/.zshrc)", "Edit(~/.claude/settings.json)", "Bash(nc *)", "Bash(wget *)" ], "allow": [ "Bash(git *)", "Bash(python3 *)", "Bash(node *)", "Bash(npm *)", "Bash(curl *)" ] }, "defaultMode": "auto" }
🚨
settings.json の編集は必ず Cursor か VS Code で：ターミナルからコピペすると改行が入ってJSONが壊れることがあります。エディタで直接編集してください。
🔔

Step 5 — Hooks の設定（完了通知・シークレット検知）

Chapter 2 · コア機能の設定
1

hooks フォルダを作成

mkdir -p ~/.claude/hooks
2

完了通知スクリプトを作成

🍎 Mac
~/.claude/hooks/notify.sh#!/bin/bash osascript -e 'display notification \ "作業完了しました" \ with title "Claude Code" \ sound name "Glass"'
🪟 Windows (WSL2)
~/.claude/hooks/notify.sh#!/bin/bash powershell.exe -Command \ "Write-Host 'Claude Code: 作業完了'"
3

実行権限を付与 & settings.json に追記

chmod +x ~/.claude/hooks/notify.sh
settings.json に追記"hooks": { "Stop": [ { "type": "command", "command": "bash ~/.claude/hooks/notify.sh" } ] }

Step 6 — Skills（スラッシュコマンド）の作成

Chapter 2 · コア機能の設定

Step 7 — Memory システムの初期設定

Chapter 2 · コア機能の設定

Memory はセッションをまたいで情報を記憶します。「覚えておいて」と話しかけるだけで自動保存されます。

1

「覚えておいて」と話しかけるだけで自動保存

# 好みを記憶させる > コードのコメントは最小限にしてください。毎回この指示を覚えておいて。 # 参照情報を記憶させる > バグ管理は GitHub Issues の「backend」プロジェクトです。覚えておいて。 # ルールを記憶させる > 本番DBには直接接続しないこと。必ずステージングで試してから。覚えておいて。
2

rules フォルダで業務ルールを体系化

~/.claude/rules/content.md（例）# SNS・コンテンツ作成ルール - 一人称は「私」 - 体言止めを多用する - 専門用語は使わず平易な言葉で - X投稿は絵文字2個まで、140文字基本
💡
フィードバックループが品質を上げる鍵：Claudeが間違えたとき → なぜ間違えたか分析 → rules/ に追記 → 同じミスが繰り返されなくなる。このサイクルを続けると数ヶ月後には自分専用のAIに育ちます。

よくあるトラブルと対処法

Appendix · トラブルシューティング

❌ command not found: claude

Node.js のグローバルパスが通っていない。

npm config get prefix # 表示されたパス/bin を PATH に追加 # ~/.zshrc に追記： export PATH="$HOME/.npm-packages/bin:$PATH"

❌ MCP が繋がらない / ツールが表示されない

settings.json の JSON 形式が壊れている可能性。

python3 -m json.tool ~/.claude/settings.json # エラーが出なければ JSON は正常

❌ Hook が実行されない

chmod +x ~/.claude/hooks/*.sh bash ~/.claude/hooks/notify.sh # 手動で動くか確認

❌ 定期実行が動かない（Mac launchd）

launchctl unload ~/Library/LaunchAgents/com.myname.digest.plist launchctl load ~/Library/LaunchAgents/com.myname.digest.plist cat ~/work/logs/digest.log # ログで確認

❌ Windows で日本語が文字化けする

sudo locale-gen ja_JP.UTF-8 sudo update-locale LANG=ja_JP.UTF-8

このガイドを完了したら

インストール → CLAUDE.md → セキュリティ → Hooks → Skills → Memory → MCP → 定期実行の順に設定することで、Claude Code が「自分専用の AI エージェント」として動き始めます。最初の設定に時間をかけた分、毎日の業務が自動化されていきます。
