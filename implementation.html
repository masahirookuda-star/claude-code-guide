Step 7 — Memory システムの初期設定

Chapter 2
📋 コピペOK — claude に貼り付けて初期メモリを一括登録（claude起動後に貼り付け）
以下の情報をすべてメモリに保存してください。 【フィードバック】 - コードのコメントは最小限にする（変数名・関数名で意味を伝える） - 返答は短く要点のみ。長文にしない - ファイルを削除・上書きする前に必ず確認する - git push する前に必ず確認する - settings.json の編集は Cursor/VS Code で行う（コピペ改行問題を防ぐため） - 2行以上のコマンドはスクリプトファイルに書いて bash で実行する - APIキーはチャットに貼らず、ファイルに書いてパスを伝える 【参照情報】 - スクリプトの置き場所：~/work/scripts/ - ログの置き場所：~/work/logs/ - 業務ファイル：~/work/docs/ これらをメモリファイルとして ~/.claude/ 配下に保存してください。
📋 コピペOK — Rules フォルダと基本ルールを一括作成（ターミナルに貼り付けて実行）
mkdir -p ~/.claude/rules cat > ~/.claude/rules/content.md << 'EOF' # SNS・コンテンツルール - 一人称は「私」、体言止めを多用する、専門用語は使わない - X投稿：絵文字2個まで・140文字基本 / note記事：冒頭3行で結論 EOF cat > ~/.claude/rules/ops.md << 'EOF' # 業務・運用ルール - 本番DBには直接接続しない（ステージングで確認後） - APIキーはファイル経由で渡す（チャットに貼らない） - 定期実行スクリプトはログを必ず記録する EOF echo "Rules setup complete!"

Step 9 — 定期自動実行の設定

📋 コピペOK — 汎用実行スクリプト（~/work/scripts/run-claude.sh）

#!/bin/bash LOG=~/work/logs/claude_auto.log && mkdir -p ~/work/logs echo "[$(date '+%Y-%m-%d %H:%M')] 開始" >> "$LOG" claude --dangerously-skip-permissions -p " # ここに実行したいプロンプトを書く 今日の組織マネジメント関連の最新情報を5件収集し各100文字以内で要約。 Slack MCPで #knowledge に投稿。冒頭：「本日のナレッジ ($(date +%Y/%m/%d))」 " >> "$LOG" 2>&1 echo "[$(date '+%Y-%m-%d %H:%M')] 完了" >> "$LOG"
🍎 Mac — launchd（毎朝7時）
📋 コピペOK — plist（[yourname]を書き換えて ~/Library/LaunchAgents/ に保存）
<plist version="1.0"><dict> <key>Label</key><string>com.yourname.claude-auto</string> <key>ProgramArguments</key><array> <string>/bin/bash</string> <string>/Users/[yourname]/work/scripts/run-claude.sh</string> </array> <key>StartCalendarInterval</key> <dict><key>Hour</key><integer>7</integer> <key>Minute</key><integer>0</integer></dict> </dict></plist>

🪟 Windows — タスクスケジューラ（PowerShell管理者）
📋 コピペOK — PowerShell（管理者権限で実行）
$a = New-ScheduledTaskAction -Execute "wsl.exe" -Argument "bash ~/work/scripts/run-claude.sh" $t = New-ScheduledTaskTrigger -Daily -At "07:00" Register-ScheduledTask -TaskName "ClaudeAutoRun" -Action $a -Trigger $t -RunLevel Highest -Force Write-Host "登録完了"

// スクリプトプロパティに設定： // GEMINI_API_KEY / SLACK_BOT_TOKEN / SLACK_USER_ID / MEETING_KEYWORDS(商談,MTG,打合せ,面談) function generateBriefs() { var p = PropertiesService.getScriptProperties(); var keywords = (p.getProperty('MEETING_KEYWORDS') || '商談,MTG,打合せ').split(','); var cal = CalendarApp.getDefaultCalendar(); var tomorrow = new Date(); tomorrow.setDate(tomorrow.getDate() + 1); cal.getEventsForDay(tomorrow).forEach(function(ev) { var title = ev.getTitle(); if (!keywords.some(function(kw){return title.indexOf(kw)!==-1;})) return; var res = UrlFetchApp.fetch( 'https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent?key='+p.getProperty('GEMINI_API_KEY'), { method:'POST', contentType:'application/json', payload: JSON.stringify({contents:[{parts:[{text: title+'の商談相手について①会社概要②最近のニュース③想定課題④話すべきポイントを日本語500字以内でまとめてください。' }]}]}) } ); var brief = JSON.parse(res.getContentText()).candidates[0].content.parts[0].text; ev.setDescription('【AI商談ブリーフ】\n' + brief); var ch = JSON.parse(UrlFetchApp.fetch('https://slack.com/api/conversations.open', { method:'POST', headers:{Authorization:'Bearer '+p.getProperty('SLACK_BOT_TOKEN')}, contentType:'application/json', payload:JSON.stringify({users:p.getProperty('SLACK_USER_ID')}) } ).getContentText()).channel.id; UrlFetchApp.fetch('https://slack.com/api/chat.postMessage', { method:'POST', headers:{Authorization:'Bearer '+p.getProperty('SLACK_BOT_TOKEN')}, contentType:'application/json', payload: JSON.stringify({channel:ch, text:'📋 明日の商談ブリーフ：'+title+'\n\n'+brief}) }); }); } // トリガー設定：時間主導型 → 日付ベース → 午前6〜7時

よくあるトラブルと対処法

❌ command not found: claude

npm config get prefix # 表示パス/bin を ~/.zshrc に追加 export PATH="$HOME/.npm-packages/bin:$PATH"

❌ settings.json が読み込まれない

python3 -m json.tool ~/.claude/settings.json # エラー→Cursor/VS Codeで再編集

❌ Hook が動かない

chmod +x ~/.claude/hooks/*.sh bash ~/.claude/hooks/notify.sh

❌ launchd が動かない（Mac）

launchctl unload ~/Library/LaunchAgents/*.plist launchctl load ~/Library/LaunchAgents/*.plist

❌ MCP ツールが見えない

claude を再起動。settings.json の mcpServers セクションのJSON形式を確認。

❌ Windows 日本語化け

sudo locale-gen ja_JP.UTF-8 sudo update-locale LANG=ja_JP.UTF-8

実装完了後にやること — フィードバックループで育てる

Claudeが間違えたとき → なぜ間違えたか分析 → rules/ や CLAUDE.md に追記 → 同じミスが繰り返されなくなる。このサイクルが数ヶ月後に大きな差を生みます。

最終チェックリスト

  • Node.js と Claude Code をインストールした
  • claude.ai の Pro 以上のアカウントでログインした
  • ~/.claude/CLAUDE.md にプロフィール・ルールを書いた
  • settings.json にセキュリティ設定（deny/allow）を追加した
  • 完了通知フック（Stop hook）を設定してテストした
  • Skills 6種類を一括作成した
  • 「コピペ初期メモリ」を claude に貼り付けて記憶させた
  • Slack / Gmail / Googleカレンダーのどれかを MCP 接続した
  • 定期実行スクリプトを1本作成してスケジューラに登録した
  • Cursor をインストールして Claude Code と使い分けを把握した
🎉
全項目チェックできたら完了！ Rules にフィードバックを蓄積し続けることで、Claude Code はどんどん「自分のことを知っているAI」に育っていきます。
